第一次写算法相关的帖子,在写最小生成树之前先写一下并查集(DSU)。

## 并查集概念

### 什么是并查集

并查集是用来处理不交集的合并与查询问题的数据结构。
并查集支持两类操作：
    1. Find(查询)
        找到某元素属于哪个集合。
        如 `find(x)` 可以找到 `x` 所在集合的根节点。
    2. Union(合并)
        将两个集合合并成一个集合。
        如 `union(x, y)` 可以合并 `x` 和 `y` 所在的集合。

### 并查集的核心思想

把每个集合看成一棵树，树的根节点代表这个集合，每个节点都有自己的父节点，每个节点也可以利用 find 函数找到自己的根节点。
为了效率，这里有两个技巧：
    1. 路径压缩
        `find` 函数执行时，如果找到了 `x` 的根节点，就把 `parent[x]` 设为找到的根节点，避免二次查找。
    2. 按大小合并
        合并两棵树的时候，把小树挂在大树下面，避免树过深，不利于查找。

## 并查集代码

### 原代码

```cpp
struct DSU
{
    // parent 是节点对应的父节点，cnt 是节点所在集合中总节点数;
    vector<int> parent, cnt;
    // 初始化 n 个节点，每个节点自成一个集合
    DSU(int n)
    {
        parent.resize(n);
        cnt.resize(n, 1);
        for(int i = 0; i < n; i++)
        {
            parent[i] = i; // 每个节点的父节点默认是自己
        }
    }
    // 查询 x 所在集合的根节点
    int find(int x)
    {
        if(x == parent[x]) // 父节点是他自身，说明这是独立的一个点或者说是根节点
        {
            return x;
        }
        return parent[x] = find(parent[x]); // 利用递归查找根节点
    }
    // 合并 x 和 y 所在的集合
    bool unite(int x, int y)
    {
        x = find(x);
        y = find(y);
        if(x == y)
        {
            return false; // 已在同一集合，不能合并
        }
        // 按大小合并
        if(cnt[x] < cnt[y])
        {
            swap(x, y);
        }
        parent[y] = x;
        cnt[x] += cnt[y];
        return true;
    }
};
```

### 使用示例

```cpp
int main()
{
    DSU dsu(5); // 0,1,2,3,4
    dsu.unite(0, 1);
    dsu.unite(1, 2);
    dsu.unite(3, 4);
    cout << (dsu.find(0) == dsu.find(2)) << "\n"; // 1，0 和 2 在同一个集合
    cout << (dsu.find(0) == dsu.find(3)) << "\n"; // 0，不同集合
    return 0;
}
```

## 并查集用途

并查集算是很基本的一个数据结构，很多地方都会用到：
    1. 判断图的连通性
        ``unite(u,v)`` 合并边连接的点。
        判断两个点是否连通：``find(u)==find(v)``。
    2. 最小生成树（Kruskal）
        遍历边，``unite(u,v)`` 只有不在同一集合才加入 MST。
    3. 检测环
        遍历边，如果 ``find(u)==find(v)``，说明加这条边会成环。
    4. 动态连通性
        处理合并和查询集合的问题，效率非常高。
